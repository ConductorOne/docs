C1 provides identity governance for Productboard. Integrate your Productboard
instance with C1 for unified visibility and governance over user access.

## Capabilities

|Resource|Sync|Provision|
|-|-|-|
|Members|<Icon icon="square-check" iconType="solid" color="#c937ae"/>||

The connector syncs your Productboard workspace **members** as user identities.
Each member carries their workspace **role** — `admin`, `maker`, `contributor`,
or `viewer` — as a profile attribute. The access model is flat: roles are
attributes on the member, so the connector emits no separate entitlements or
grants.

<Warning>
This connector uses **Productboard API v1**, which Productboard has announced
will be retired on **July 8, 2026**. After that date the connector will stop
syncing until it is migrated to Productboard API v2. Plan to update before the
sunset date.
</Warning>

## Gather Productboard credentials

<Warning>
You need Productboard **workspace administrator** access to create an API access
token.
</Warning>

<Steps>
  <Step>
    In Productboard, go to **Settings** > **Integrations** > **Public API**.
  </Step>

  <Step>
    Generate an **Access Token**. The token reads the members of the workspace
    it is created in; the connector only ever reads (it never writes).
  </Step>

  <Step>
    Copy the generated token — you will provide it when configuring the
    connector.
  </Step>
</Steps>

## Configure the Productboard connector

<Tabs>
  <Tab title="Cloud-hosted">
    Follow these instructions to use a built-in, no-code connector hosted by C1.

    <Steps>
      <Step>
        In C1, navigate to **Integrations** > **Connectors** and click **Add connector**.
      </Step>

      <Step>
        Search for **Productboard** and click **Add**.
      </Step>

      <Step>
        Choose how to set up the new Productboard connector.
      </Step>

      <Step>
        Set the owner for this connector.
      </Step>

      <Step>
        Click **Next**.
      </Step>

      <Step>
        Find the **Settings** area of the page and click **Edit**.
      </Step>

      <Step>
        Enter the Productboard credentials:

        - **Productboard API Token**: The access token you created.
      </Step>

      <Step>
        Click **Save**.
      </Step>

      <Step>
        The connector's label changes to **Syncing**, followed by **Connected**. You can view the logs to ensure that information is syncing.
      </Step>
    </Steps>

    **Done.** Your Productboard connector is now pulling access data into C1.
  </Tab>

  <Tab title="Self-hosted">
    Follow these instructions to run the Productboard connector in your own
    environment.

    <Steps>
      <Step>
        Create a secret for the Productboard access token.
      </Step>

      <Step>
        Configure the connector environment variables:

        - **PRODUCTBOARD_API_TOKEN**: The access token you created (store this as a secret).
      </Step>

      <Step>
        Deploy the connector using your standard self-hosted connector process.
      </Step>
    </Steps>

    **Done.** Your Productboard connector is now pulling access data into C1.
  </Tab>
</Tabs>
