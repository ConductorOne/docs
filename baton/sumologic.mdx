C1 provides identity governance for Sumo Logic. Integrate your Sumo Logic
account with C1 for unified visibility and governance over user access.

## Capabilities

|Resource|Sync|Provision|
|-|-|-|
|Users|<Icon icon="square-check" iconType="solid" color="#c937ae"/>||

The connector syncs the users of your Sumo Logic account as identities. Each
user carries their assigned role IDs, so you can see who has access to your Sumo
Logic account and which roles grant that access.

## Gather Sumo Logic credentials

<Warning>
You need a Sumo Logic **access key** (an access ID and access key pair) created
by an administrator with permission to view users and roles. The access key
grants API access to your account, so treat both halves as secrets.
</Warning>

<Steps>
  <Step>
    Sign in to Sumo Logic as an administrator of the account you want to sync.
  </Step>

  <Step>
    Open **Administration** > **Security** > **Access Keys** and click **Add
    Access Key** (or create a personal key under **Preferences** > **My Access
    Keys**). Give it a name you will recognize.
  </Step>

  <Step>
    Copy the **Access ID** and **Access Key** when they are shown — the access
    key is displayed only once and cannot be retrieved later.
  </Step>

  <Step>
    Note your deployment's **API base URL** under **Administration** >
    **Account** > **API URLs**. It depends on your region, for example
    `https://api.sumologic.com` (US1), `https://api.us2.sumologic.com` (US2),
    `https://api.eu.sumologic.com` (EU), or `https://api.au.sumologic.com` (AU).
  </Step>
</Steps>

## Configure the Sumo Logic connector

<Tabs>
  <Tab title="Cloud-hosted">
    Follow these instructions to use a built-in, no-code connector hosted by C1.

    <Steps>
      <Step>
        In C1, navigate to **Integrations** > **Connectors** and click **Add connector**.
      </Step>

      <Step>
        Search for **Sumo Logic** and click **Add**.
      </Step>

      <Step>
        Choose how to set up the new Sumo Logic connector.
      </Step>

      <Step>
        Set the owner for this connector.
      </Step>

      <Step>
        Click **Next**.
      </Step>

      <Step>
        Find the **Settings** area of the page and click **Edit**.
      </Step>

      <Step>
        Enter the Sumo Logic credentials:

        - **API base URL**: Your deployment's regional API host, for example
          `https://api.us2.sumologic.com`.
        - **Access ID**: The access ID from your access key pair.
        - **Access key**: The access key from your access key pair.
      </Step>

      <Step>
        Click **Save**.
      </Step>

      <Step>
        The connector's label changes to **Syncing**, followed by **Connected**. You can view the logs to ensure that information is syncing.
      </Step>
    </Steps>

    **Done.** Your Sumo Logic connector is now pulling access data into C1.
  </Tab>

  <Tab title="Self-hosted">
    Follow these instructions to run the Sumo Logic connector in your own
    environment.

    <Steps>
      <Step>
        Create secrets for the Sumo Logic access ID and access key.
      </Step>

      <Step>
        Configure the connector environment variables:

        - **BATON_SUMOLOGIC_BASE_URL**: Your deployment's regional API host (for example `https://api.us2.sumologic.com`).
        - **BATON_SUMOLOGIC_ACCESS_ID**: The access ID from your access key pair.
        - **BATON_SUMOLOGIC_ACCESS_KEY**: The access key from your access key pair (store this as a secret).
      </Step>

      <Step>
        Deploy the connector using your standard self-hosted connector process.
      </Step>
    </Steps>

    **Done.** Your Sumo Logic connector is now pulling access data into C1.
  </Tab>
</Tabs>
