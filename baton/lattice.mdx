C1 provides identity governance for Lattice. Integrate your Lattice instance
with C1 for unified visibility and governance over user access.

## Capabilities

|Resource|Sync|Provision|
|-|-|-|
|Users|<Icon icon="square-check" iconType="solid" color="#c937ae"/>||
|Departments|<Icon icon="square-check" iconType="solid" color="#c937ae"/>||

The connector syncs your Lattice users as identities. Each user carries their
job title, department, and account status, so you can see who is active and
where they sit in the organization. It also syncs your Lattice departments and
who belongs to each one, so department membership is visible and reviewable in
C1.

## Gather Lattice credentials

<Warning>
You need a Lattice **API key** created by a Lattice **admin**. The API key
grants access to your Lattice account's user and department data, so treat it as
a secret.
</Warning>

<Steps>
  <Step>
    Sign in to Lattice as an administrator.
  </Step>

  <Step>
    Go to **Admin** > **Platform** > **API keys** and click **Generate API
    key**.
  </Step>

  <Step>
    Copy the API key immediately — Lattice shows it only once. You will use it to
    configure the connector.
  </Step>
</Steps>

## Configure the Lattice connector

<Tabs>
  <Tab title="Cloud-hosted">
    Follow these instructions to use a built-in, no-code connector hosted by C1.

    <Steps>
      <Step>
        In C1, navigate to **Integrations** > **Connectors** and click **Add connector**.
      </Step>

      <Step>
        Search for **Lattice** and click **Add**.
      </Step>

      <Step>
        Choose how to set up the new Lattice connector.
      </Step>

      <Step>
        Set the owner for this connector.
      </Step>

      <Step>
        Click **Next**.
      </Step>

      <Step>
        Find the **Settings** area of the page and click **Edit**.
      </Step>

      <Step>
        Enter the Lattice credentials:

        - **API key**: The API key you created in Lattice.
      </Step>

      <Step>
        Click **Save**.
      </Step>

      <Step>
        The connector's label changes to **Syncing**, followed by **Connected**. You can view the logs to ensure that information is syncing.
      </Step>
    </Steps>

    **Done.** Your Lattice connector is now pulling access data into C1.
  </Tab>

  <Tab title="Self-hosted">
    Follow these instructions to run the Lattice connector in your own
    environment.

    <Steps>
      <Step>
        Create a secret for the Lattice API key.
      </Step>

      <Step>
        Configure the connector environment variables:

        - **BATON_LATTICE_API_KEY**: The Lattice API key (store this as a secret).
      </Step>

      <Step>
        Deploy the connector using your standard self-hosted connector process.
      </Step>
    </Steps>

    **Done.** Your Lattice connector is now pulling access data into C1.
  </Tab>
</Tabs>
