## How does C1's helpdesk automation work? 

If your organization’s IT service desk wants to keep its current processes and toolchain, but needs to automate approval and provisioning workflows on the backend, C1’s AI-powered helpdesk automation can manage approval routing and provisioning behind the scenes.

When your colleague request access by creating a service desk ticket, C1 automatically turns the information in the ticket into a request task. As the request task proceeds through the approval and provisioning steps in C1, it updates the Jira ticket, maintaining a full audit trail. 

## Integrate your Jira Service Management instance

<Tip>
**C1 currently supports Jira Service Management.** 

The instructions below set up an integration with **Jira Service Management** that allows C1 to create, modify, and resolve Jira service desk tickets. 

If you want to connect a **Jira Cloud** instance with C1 so that you can review Jira access data and grant user access to Jira, go to [Set up a Jira Cloud connector](/baton/jira).
</Tip>

The integration process includes setup tasks in both C1 and Jira Service Management. We'll guide you through the process.  

In C1:

<Steps>
<Step>
Navigate to **Integrations** > **Service desk**.
</Step>

<Step>
Click **Add integration**. 
</Step>

<Step>
Select your service desk provider and set a display name for the integration.
</Step>

<Step>
Click **Save**. Next, you'll need to install the C1 app in Jira Service Management. 
</Step>
</Steps>

In Jira Service Management: 

<Steps>
<Step>
Click **Apps** > **Connected apps** > **Settings**. 
</Step>
<Step>
Enable **Developer mode**.
</Step>
<Step>
Click **Upload app**. 
</Step>

<Step>
In the **From this URL** field, enter the following: 

   ```
   https://marketplace.atlassian.com/files/1.1.0-AC/e87e112f-5c07-462d-a57a-d3cd0fd3840f
   ```
</Step>
<Step>
Click **Upload**. 
</Step>
<Step>
Finally, copy the base URL of your Jira Service Management instance (for example, `https://example.atlassian.net`). 
</Step>
</Steps>

Back in C1: 

<Steps>
<Step>
Click **Edit** and paste the base URL of your Jira Service Management instance into the **URL** field. 
</Step>
<Step>
Click **Done**. 
</Step>
</Steps>

## Configure AI request settings 

Once the integration is set up, you'll see the option to configure how it will work in your Jira Service Management instance. 

<Steps>
<Step>
Click **Edit** to open the editing panel. 
</Step>
<Step>
To enable C1 to create, update, and resolve Jira Service Desk tickets for you, click to turn on **Activated**. 

<Tip>
   **Set up automation to add the `c1` label.**
   
   When activated, C1 will automatically process all Jira tickets with a `c1` label in the project you specify as access request tickets. Make sure to set up automation in Jira Service Management to add this label to the appropriate tickets. 
</Tip>
</Step>
</Steps>


<Steps>
<Step>
In the **Jira project** field, select the project where you want C1 to process tickets. 
</Step>
<Step>
**Optional.** If you want C1 to only act on tickets from a specific request type, select it from the **Jira request type** dropdown.
</Step>
<Step>
To turn on automatic submission of any valid access request processed by C1, enable **Auto-submit requests**. Invalid requests will still be routed to a member of your organization for manual review and processing.  

   If you leave this option disabled, all requests will be submitted for manual review before proceeding. 
</Step>
<Step>
**Optional.** In the **Field level hinting** section, help C1 correctly extract the information from Jira tickets to create access requests by mapping C1 access request attributes to the ticket fields in Jira where the information should be pulled from. 

   <Tip>
      **Field type limitations.** 
      
      C1 can only pull data from custom Jira fields of these types: 

      * Single select
      * Radio buttons
      * Single line text
      * Cascading select

      Jira ticket fields that use a different field type cannot be used, and will not be shown in the ticket field selection dropdown. 
   </Tip>
</Step>
</Steps>

<Steps>
<Step>
In the **Jira ticket status for resolved requests** field, select the Jira ticket status that you want C1 to automatically set when a request ticket is complete. 
</Step>
<Step>
**Optional.** In the **Resolved state mapping** section, set how you want C1 closed request task status to map to Jira ticket resolution states. 

   C1 sets each closed request task to one of the following statuses:

   * Granted (the access was approved and provisioned)
   * Denied (the access was not approved)
   * Canceled
</Step>
<Step>
Finally, in the **Flagging tickets** section, set a label that C1 will automatically apply to Jira tickets in need of manual intervention. 

   Set up filters or workflows in Jira Service Management to pull tickets with this label into your team's work queue. 
</Step>
<Step>
When you've finished making your selections, click **Save**. 
</Step>
</Steps>

**Done.** Your service desk integration is ready to use.
