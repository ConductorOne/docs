C1 uses single sign-on (SSO) for user login. Instead of a separate C1 password, your users authenticate through your existing identity provider (IdP), so access to C1 follows the same lifecycle, security policies, and offboarding as the rest of your apps.

## Supported identity providers

You connect C1 to your IdP using OpenID Connect (OIDC). C1 documents step-by-step login setup for the following providers:

| Identity provider | Protocol |
| :--- | :--- |
| [Okta](/product/how-to/qs-set-up-c1#authenticate-with-okta) | OIDC |
| [Google](/product/how-to/qs-set-up-c1#authenticate-with-google) | OIDC |
| [Microsoft Entra ID (Azure AD)](/product/how-to/qs-set-up-c1#authenticate-with-microsoft) | OIDC |
| [OneLogin](/product/how-to/qs-set-up-c1#authenticate-with-onelogin) | OIDC |
| [JumpCloud](/product/how-to/qs-set-up-c1#authenticate-with-jumpcloud) | OIDC |
| [PingOne](/product/how-to/qs-set-up-c1#authenticate-with-pingone) | OIDC |
| [Any OpenID Connect (OIDC) provider](/product/how-to/qs-set-up-c1#authenticate-with-generic-openid-connect) | OIDC |

## Set up SSO

You configure SSO when you create your C1 tenant. During setup, you choose your identity provider, register C1 as an application in that provider, and exchange the OIDC client credentials so users can sign in.

For complete, per-provider instructions — including how to create the C1 application in your IdP and where to paste the client ID and client secret — see [Create a C1 tenant](/product/how-to/qs-set-up-c1).

<Steps>
  <Step>
    Choose your identity provider (Okta, Google, Microsoft, OneLogin, JumpCloud, PingOne, or a generic OIDC provider).
  </Step>
  <Step>
    In your identity provider, create an application for C1 and assign the users or groups who should be able to sign in.

    <Warning>
    Assign the C1 application to yourself before you finish setup. If you do not, you will receive an error at login and your C1 tenant will not be created.
    </Warning>
  </Step>
  <Step>
    Copy the OIDC client ID and client secret from your identity provider and paste them into the matching fields in C1.
  </Step>
  <Step>
    Click **Sign up with [your provider]** to complete SSO setup and sign in.
  </Step>
</Steps>

## Manage SSO sessions

After SSO is set up, you can tune how long sessions last and where they can be used:

- **Session length** — control how long a C1 session stays valid before users must re-authenticate. See [Configure session length](/product/admin/global-settings#configure-session-length).
- **IP allow lists for SSO sessions** — restrict SSO sessions (and API keys) to trusted IP ranges. See [Configure global IP allow lists](/product/admin/global-settings#configure-global-ip-allow-lists).
- **Step-up authentication** — require additional authentication for sensitive approvals. See [Step-up authentication](/product/admin/step-up-auth).
